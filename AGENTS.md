# BMad Method

This project uses the BMad Method lifecycle. Skills are invoked via `/bmad-<skill-name>`. Overrides live in `_bmad/custom/`. Config resolves from `_bmad/config.toml` + `_bmad/custom/config.toml`.

## Skill categories by lifecycle phase:
- **Analysis (Phase 1):** product-brief, prfaq, agent-analyst, agent-tech-writer, document-project, domain/market/technical research
- **Planning (Phase 2):** prd, ux, agent-pm, agent-ux-designer, create/edit/validate-prd
- **Solutioning (Phase 3):** architecture, agent-architect, create-architecture, create-epics-and-stories, check-implementation-readiness, generate-project-context
- **Implementation (Phase 4):** dev-story, quick-dev, agent-dev, code-review, sprint-planning/status, create-story, correct-course, qa-generate-e2e-tests, retrospective, checkpoint-preview

## Core utilities
- **brainstorming** — creative sessions (facilitator / partner / autonomous)
- **party-mode** — multi-agent roundtables
- **help** — skill catalog and advice
- **advanced-elicitation** — deeper probing when needed
- **spec** — specification authoring
- **forge-idea** — idea development
- **shard-doc** — document splitting
- **editorial-review-structure/prose** — polish passes
- **adversarial-general / edge-case-hunter** — review lenses
