# Epic 3 — Water Animation: Retrospective

**Status:** Complete

## Summary

All 4 stories delivered — WaterFill component with SVG wave, ripple on drink log, count-up number animation, and goal-reached celebration. The water animation became the app's signature visual. Significant post-epic refinement happened during the HomeScreen redesign (glass-shaped clip path, flat surface mode, dark mode).

## What Went Well

- **Reanimated-based UI-thread animation** — the wave runs at 60fps entirely on the UI thread via worklets. No JS thread jank even on low-end Android devices.
- **Layer-based depth illusion** — two sine wave layers at different speeds/opacities created a convincing water-like appearance that was visually premium.
- **Goal-reached moment** (3.4) — the overshoot/shimmer/"slosh" effect made hitting the goal feel genuinely rewarding. Combined with AchievementPopup confetti, it was the right punch.
- **Smooth fill transitions** — fill changes tween smoothly via `withTiming` (600ms ease-out) instead of snapping.
- **Ripple on log** (3.2) — Gaussian amplitude bump timed to the drink log action gave satisfying real-time feedback.

## What Could Be Better

- **Wave animation complexity** — the two-layer sine system produced spike artifacts when clipped to the glass shape, requiring post-epic fixes (background rect, then later full amplitude=0 flat mode per user request).
- **Over-engineering** — the continuous wave motion and ripple were removed in post-epic cleanup because the user preferred a clean flat surface. A simpler approach from the start would have saved rework.
- **water-clip path duplication** — the glass clip path is defined inline in HomeScreen.js and referenced by name. It worked but felt fragile compared to a shared SVG def.
- **Glass outline SVG** — the glass outline uses a separate inline Path that duplicates the clip path math. A single source of truth would reduce visual mismatch risk.
- **Spike artifacts** — took several rounds to fully eliminate the vertical white line artifacts at the clip boundary, which were the most visible user-facing bug post-launch.

## Action Items

- [ ] Kill unused Reanimated hooks (bodyProps, shimmerProps) removed in post-epic cleanup — confirm nothing else depends on them
