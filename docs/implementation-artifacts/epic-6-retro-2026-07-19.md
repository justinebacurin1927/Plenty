# Epic 6 — Mascot & Micro-interactions: Retrospective

**Status:** Complete

## Summary

All 4 stories delivered — mascot idle animation, press feedback + haptics, screen transitions + toast, weekly chart entrance. Every story went through code review, with ~20 issues caught and fixed across all reviews. 243-244 tests passing.

## What Went Well

- **Code review rigor caught real bugs** — 6.4 review found 2 crash-level NaN bugs (empty data, array mismatch) that would have hit production. 6.3 review caught timer leaks, stale callbacks, and missing `pointerEvents`.
- **Cross-cutting patterns mastered** — `useReducedMotion` consistently applied in every story. `Animated.spring` configs (friction/tension) standardized. Legacy `Animated` API used correctly per AD-11.
- **Animation cleanup discipline** — By 6.4, all animations had proper `stopAnimation()` cleanup on unmount and effect re-runs, learned from 6.3's Toast review.
- **Edge cases hardened** — Empty state handling (`maxTotal=0`), index bounds checks, timer cleanup, null-safe accessors (`??` pattern).
- **Test coverage maintained** — No regressions introduced across 20+ test files. Every new component (Toast, PressableScale) got its own test file.

## What Could Be Better

- **Epic 7 dependency not started** — Epic 6.1 lists Epic 7 (Design System) as a dependency for tokens, but Epic 7 is still backlogged. Tokens would have made visual consistency easier across components.
- **6.1 review had deferred items** — 2 review findings deferred (idle bob pause on tap, blink timer cleanup). Low severity but worth revisiting.
- **streak-flame test teardown timeout** — Pre-existing flaky test that produces a noisy "Jest did not exit" warning. Minor but annoying.
