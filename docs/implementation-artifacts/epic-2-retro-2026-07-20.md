# Epic 2 — Streak System: Retrospective

**Status:** Complete

## Summary

All 7 stories delivered — streak engine, heatmap, rewards ladder, streak freeze, streak flame, streak on widget, and streak-aware notifications. The streak system became the core emotional hook of the app with full persistence, visual feedback, and widget integration.

## What Went Well

- **Comprehensive test coverage** — streak math, freeze behavior, and history edge cases all unit-tested. No regressions introduced across the 7-story run.
- **Widget integration** (2.6) — delivered a functioning Android home screen widget showing current streak, which required crossing the React Native / native Kotlin boundary successfully.
- **Streak-aware notifications** (2.7) — variant notification copy based on streak state kept the feature from being a static badge; it adapts to user behavior.
- **Heatmap** (2.2) — GitHub-style visual history was a high-impact feature that made streaks feel earned and visible.
- **Rewards ladder** (2.3) — milestones at 7/30/60/100/365 gave clear progression goals.

## What Could Be Better

- **Streak flame animation** (2.5) — has a flaky test teardown timeout (`--detectOpenHandles` needed) that produces noise in CI. Pre-existing, not introduced by this epic.
- **Streak freeze usage is invisible** — users can't easily see how many freezes they have remaining without digging. A small UI indicator on the streak display would help.
- **Widget show/hide timing** — the widget shows the streak based on the previous day's completion, which can feel a beat behind for users who log first thing in the morning.

## Action Items

- [ ] Fix streak-flame teardown timeout — add `--detectOpenHandles` to find the leak
- [ ] Add freeze count badge to streak display on home screen
- [ ] Look into widget streak sync timing for early-morning loggers
