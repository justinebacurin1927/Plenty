- (decision) Mascot style: Classic droplet with stubby limbs — round drop body/head, small expressive arms and legs. Playful + charming, not detailed/cartoony. Holds props (umbrella, water glass) and does full-body animations (wave, dance, stretch).
- (decision) Mascot rendering: sprite-based (PNG assets generated externally) + Reanimated transitions between states. Props as separate overlay layers. Assets in assets/mascot/ organized by category (base, weather, time, achievements, props).
- (decision) Visual language: 'Balanced Charm' — clean, restrained UI serves as stage for mascot + water animation as the showpiece moments. Contrast makes delight land harder.
- (decision) WaterFill visualization: Glass/bottle fill with animated SVG wave rising bottom-to-top. Contained within a glass outline shape that complements the droplet mascot on the Home screen.
- (decision) Home screen layout C: Mascot-led hero with speech bubble → progress bar → quick-log button → reminder controls. Glass animation lives as a secondary touch point (Log screen or celebration).
- (decision) Mascot states: weather (hot/rainy/snow/normal), time of day (morning/day/night), user actions (log/goal/streak), escalation, idle. Modular sprite layer approach: body (2-3) + eyes (4-5) + props (5) + effects (4) composited via Reanimated.
- (decision) WaterFill glass: appears as goal celebration animation (quick, satisfying reward). Log screen uses standard bar chart, not the animated glass fill.
- (decision) Achievements screen: trophy shelf visual — unlocked as physical objects (trophies, medals, ribbons) on a shelf display. Locked ones sit grayed out. More visual, less text.
- (decision) Typography: system fonts (San Francisco / Roboto) for functional UI. Custom rounded display font for speech bubbles, achievement titles, mascot voice moments, and app logo in limited doses.
- (decision) Onboarding: feature tour (3 slides) — mascot intro, set daily goal, set reminder interval. Fast path to active use. The mascot's personality wins during use, not onboarding.
- (decision) Dark mode mascot outfit: sunglasses during daytime dark mode, nightcap + sleepy eyes during nighttime dark mode. Adds personality and makes switching theme feel rewarding.
- (decision) Haptics: Medium intensity — subtle taps on regular actions, satisfying buzz on goal hit and achievement unlock. Skipped on rapid repeated actions to avoid fatigue.
- (decision) Mascot voice: Buddy tone (casual, friendly, simple) as default with Silly moments during celebrations and streaks.
- (event) spines finalized — reviewer findings resolved: token syntax, speech bubble alignment, celebration stacking exception, user-state priority, loading states, accessibility, Log/Settings flows, missing surface coverage, mascot expression visual specs
